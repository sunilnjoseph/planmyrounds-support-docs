Shifts are the fundamental building block of a roster in PlanMyRounds. A shift represents a defined block of clinical work — for example, a _Day SHO_, _Night Reg_, or _Weekend On-call_ — with a start time, end time, role, and location.

## Where shifts fit

<CardGroup cols={2}>
  <Card title="Shift Templates" icon="layer-group">

  </Card>

  <Card title="Rosters / Duty Plans" icon="calendar-days">

## Shift lifecycle

<Steps>
  <Step title="Define">
    Create a [shift template](/shifts/shift-templates) with start/end times, role, location, and pay/banding rules.
  </Step>
  <Step title="Assign">
    Drop the shift onto a person and date in the [roster builder](/shifts/assigning-staff).
  </Step>
  <Step title="Publish">
    [Publish the roster](/shifts/publishing-and-notifications) — staff are notified and shifts become visible in their schedule.
  </Step>
  <Step title="Adjust">
    Handle changes via [swaps, cover, or manual edits](/shifts/swaps-and-cover) — the audit trail records every change.
  </Step>
</Steps>

## Key concepts

| Concept | Description |
| --- | --- |
| **Shift** | A single block of work assigned to one user on one date. |
| **Shift template** | A reusable pattern used to create shifts consistently. |
| **Unassigned shift** | A shift that exists on the roster but has no owner — visible as a gap. |
| **Published vs. draft** | Draft shifts are only visible to admins; published shifts are visible to staff. |
| **Conflict** | When a shift overlaps another shift, approved leave, or rest-rule breach. |

<Tip>
  Always create shifts from templates where possible — it keeps rules (rest periods, banding, role permissions) consistent across your rota.
</Tip>
